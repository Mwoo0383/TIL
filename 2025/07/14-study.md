| **프로파일 다이어그램** | UML 확장을 정의할 때 사용                     |

### 2. 데이터베이스 회복기법

**a. 로그 기반 회복 기법**
> 모든 트랜잭션의 작업 내용을 로그 파일에 기록한 후, 장애 발생 시 로그를 이용해 회복

| 구분             | 설명                                             |
| -------------- | ---------------------------------------------- |
| **UNDO (롤백)**  | 트랜잭션의 작업이 완료되지 않은 경우, 로그를 참고해 **되돌림**          |
| **REDO (재실행)** | 트랜잭션이 완료되었으나 DB에 반영되지 않은 경우, 로그를 참고해 **다시 실행** |

**b. 체크포인트 기법**
> 일정 시간마다 현재까지의 변경 사항을 디스크에 기록(저장)하는 기법  
> 장애 발생 시 체크포인트 이후의 로그만 사용해 복구 → 성능 향상  
> 로그가 너무 많아지는 걸 방지함

**c. 그림자페이지 기법**
> 트랜잭션 수행 시, 원본 페이지는 건드리지 않고 복사본(그림자 페이지)을 먼저 수정함

| 특징        | 설명                                    |
| --------- | ------------------------------------- |
| 안정성 높음    | 원본은 변경되지 않으므로 장애 발생 시 **복사본만 폐기하면 됨** |
| 디스크 공간 낭비 | 복사본을 계속 유지해야 하므로 공간 소모 ↑              |

**d. 즉시 갱신 vs 지연 갱신**  

| 구분                           | 설명                                        |
| ---------------------------- | ----------------------------------------- |
| **즉시 갱신 (Immediate Update)** | 트랜잭션 수행 중에도 DB에 **바로 반영** → REDO, UNDO 필요 |
| **지연 갱신 (Deferred Update)**  | 트랜잭션 **완료 시점에만 반영** → REDO만 필요, UNDO 불필요  |

### 3. NAT(Network Address Translation)
> “외부의 공인 IP주소와 포트 주소에 해당하는 내부 IP주소를 재기록하여 라우터를 통해 네트워크 트래픽을 주고받는 기술”

### 4. 블록체인
> 블록체인은 중앙 서버 없이 거래 정보를 블록 단위로 묶어 체인처럼 연결하고, 이를 분산 네트워크에 저장하여 위·변조를 방지하는 기술이다.  
> 각 블록은 해시값을 통해 연결되며, P2P 방식으로 데이터 무결성과 신뢰성을 유지한다.

### 5. 하둡(Hadoop)
> 대용량 데이터를 여러 대의 컴퓨터(노드)에 분산 저장하고, 병렬로 처리할 수 있는 오픈소스 프레임워크

### 6. 테스트 오라클 종류
- **참 오라클**
: 모든 입력에 대해 정확한 결과를 알고 있음
- **휴리스틱 오라클**
: 대략적으로 결과 예측 가능
- **일관성 오라클**
: 같은 입력에 같은 결과를 기대
- **샘플링 오라클**
: 전체 입력 중 일부 샘플에 대해서만 기대 결과를 알고 있음

### 7. 테스트 기법 분류

- **명세 기반 기법**
> 시스템의 **기능(입력/출력)**을 기준으로 테스트 → 내부 로직 몰라도 됨

| 기법            | 설명                                          | 예시                              |
| ------------- | ------------------------------------------- | ------------------------------- |
| **동등 분할**     | 유효/무효 데이터를 \*\*그룹(등가 클래스)\*\*으로 나눠 대표값만 테스트 | 나이: 0\~120 → `-1`, `25`, `130`  |
| **경계값 분석**    | **경계 근처 값**을 집중적으로 테스트                      | 1\~100 → `0`, `1`, `100`, `101` |
| **의사결정 테이블**  | 입력 조합에 따른 **행동 규칙을 표로 정리**                  | 쇼핑몰 할인 조건                       |
| **상태 전이 테스트** | 상태 변화 기반 시스템을 테스트                           | 로그인 상태 → 로그아웃 상태 전이 등           |
| **유스케이스 기반**  | 사용자의 시나리오 기반으로 테스트                          | 장바구니 담기, 결제하기 등                 |

- **구조 기반 기법**
> 코드 내부 로직 기준으로 테스트 → 개발자가 주로 수행

| 기법          | 설명                            | 예시                           |
| ----------- | ----------------------------- | ---------------------------- |
| **문장 커버리지** | 모든 코드 라인이 **한 번 이상 실행**되게 테스트 | if문 안/밖 모두 실행                |
| **분기 커버리지** | 모든 **조건의 참/거짓 분기**를 테스트       | `if (a > b)` → true/false 모두 |
| **조건 커버리지** | 복합 조건문의 **각 조건별** 참/거짓 테스트    | `if (a > 5 && b < 10)` 각각 검증 |
| **경로 커버리지** | 가능한 **모든 실행 경로** 테스트          | 모든 if-else 조합 완전 수행          |

- **경험 기반 기법**
> 테스터의 경험/직관/추정에 따라 수행하는 테스트

| 기법           | 설명                               |
| ------------ | -------------------------------- |
| **탐색적 테스트**  | 테스트 설계 없이 직접 시스템을 사용하며 결함 탐색     |
| **체크리스트 기반** | 미리 정리된 항목으로 기능 검증                |
| **에러 추정**    | 경험적으로 **오류가 많이 발생했던 부분**만 집중 테스트 |

### 9. 정보 보안의 3요소

| 요소                        | 설명                             | 예시 문장                     | 키워드            |
| ------------------------- | ------------------------------ | ------------------------- | -------------- |

### 1. WSDL(Web Services Description Language)
> 웹 서비스의 기능, 파라미터, 프로토콜 등을 기술한 XML 기반의 기술 문서로, 서비스 제공자와 소비자 간 인터페이스를 정의

❗️ 함께 나오는 개념 - SOAP, UDDI

| 개념       | 요약 문장                         |
| -------- | ----------------------------- |
| **WSDL** | 웹 서비스의 기능과 데이터 형식을 기술한 XML 문서 |
| **SOAP** | 웹 서비스 간 데이터를 주고받는 표준 메시지 포맷   |
| **UDDI** | 웹 서비스 목록을 등록하고 검색할 수 있는 디렉터리  |

[제공자] → WSDL 작성 → UDDI에 등록  
[사용자] → UDDI에서 WSDL 찾음 → SOAP 메시지로 요청

### 3. IPC (Inter Process Communication)란?
> 운영체제에서 서로 다른 프로세스들이 데이터를 주고받기 위한 방법

📌 대표적인 IPC 방식
| 방식             | 설명                                  |
| -------------- | ----------------------------------- |
| **공유 메모리**     | 여러 프로세스가 **같은 메모리 공간**에 접근          |
| **세마포어**       | **공유 자원의 동기화/제어**를 위한 신호 기법         |
| **메시지 큐**      | 메시지를 **큐에 넣고 꺼내는 방식**의 통신           |
| **소켓(Socket)** | **네트워크 기반 통신** (TCP/IP 등으로 프로세스 연결) |
| **파이프(Pipe)**  | 한 프로세스의 출력 → 다른 프로세스의 입력으로 전달       |

### 4. 데이터 모델 구성요소 3가지
- **연산(Operation)**
: 개체 데이터 모델에서 실제 데이터를 처리하는 작업에 대한 명세
- **구조(Structure)**
: 논리 데이터 모델은 테이블 구조, 속성, 타입 등을 표현함
- **제약조건(Constraint)**
: 데이터 무결성 유지를 위한 데이터베이스의 보편적 방법

### 5. 시스템 접근 제어 방식
| 접근 제어 방식                       | 설명                                         | 권한 부여 주체    | 대표 예시                |
| ------------------------------ | ------------------------------------------ | ----------- | -------------------- |
| **DAC**                        |                                            |             |                      |
| (Discretionary Access Control) |                                            |             |                      |
| 임의 접근 제어                       | **소유자가 직접 권한을 부여/철회**하는 방식                 | 사용자         | Unix, Linux의 `chmod` |
| **MAC**                        |                                            |             |                      |
| (Mandatory Access Control)     |                                            |             |                      |
| 강제 접근 제어                       | 시스템이 **정책에 따라 강제로 통제**하는 방식<br>→ 사용자 판단 불가 | 보안 관리자      | 군사 시스템, 보안 등급        |
| **RBAC**                       |                                            |             |                      |
| (Role-Based Access Control)    |                                            |             |                      |
| 역할 기반 접근 제어                    | 사용자의 역할(Role)에 따라 접근 권한 부여                 | 관리자 (역할 설정) | 조직의 직무별 시스템          |
| **TBAC**                       |                                            |             |                      |
| (Task-Based Access Control)    |                                            |             |                      |
| 업무 기반 접근 제어                    | 특정 **작업(Task)에 따라 일시적 권한** 부여              | 시스템         | 워크플로우 기반 시스템 등       |

### 6. 결합도 종류
| 결합도 종류                        | 설명                                 | 예시                               | 강/약            |
| ----------------------------- | ---------------------------------- | -------------------------------- | -------------- |
| **내용 결합 (Content Coupling)**  | 한 모듈이 **다른 모듈 내부를 직접 참조**          | 다른 모듈 변수나 코드 조작                  | ❌ 가장 나쁨 (최악)   |
| **공통 결합 (Common Coupling)**   | **공용 전역 변수**를 여러 모듈이 함께 사용         | 전역 변수 `globalVar` 사용             | 매우 강함          |
| **외부 결합 (External Coupling)** | 모듈들이 \*\*외부 환경(파일, 프로토콜 등)\*\*을 공유 | 같은 파일, DB 등을 직접 접근               | 강함             |
| **제어 결합 (Control Coupling)**  | \*\*제어 플래그(논리값)\*\*를 전달해 흐름 제어     | `function(x, flag)` → flag 따라 분기 | 강함             |
| **스탬프 결합 (Stamp Coupling)**   |                                    |                                  |                |
| = 자료구조 결합                     | **필요하지 않은 전체 자료구조를 전달**            | 구조체 전체를 전달하지만 일부만 사용             | 중간 정도          |
| **자료 결합 (Data Coupling)**     | **필요한 데이터만** 인자로 전달                | `function(a, b)` 처럼 단순 데이터만      | ✅ 가장 이상적 (느슨함) |

### 7. 사이버 공격 기법
| 공격 기법                           | 설명                                   | 시험 키워드                    |
| ------------------------------- | ------------------------------------ | ------------------------- |
| **스니핑 (Sniffing)**              | 네트워크상 **데이터를 몰래 도청**                 | 도청, 패킷 가로채기, 비인가 감청       |
| **스푸핑 (Spoofing)**              | **신원을 위조**하여 속임                      | IP 위조, DNS 스푸핑, ARP 스푸핑   |
| **세션 하이재킹 (Session Hijacking)** | **세션을 탈취**하여 사용자인 척                  | RST 패킷, 세션 가로채기           |
| **피싱 (Phishing)**               | **가짜 사이트로 유도**하여 개인정보 탈취             | 메일, 문자, 피싱 사이트            |
| **파밍 (Pharming)**               | **정상 사이트를 위장**, 접속하면 공격자 페이지로 유도     | DNS 변조, Hosts 파일 조작       |
| **디렉토리 트래버설**                   |                                      |                           |
| (Directory Traversal)           | `../` 경로로 서버의 **허용되지 않은 상위 디렉토리 접근** | 웹서버 취약점                   |
| **XSS (Cross Site Scripting)**  | 웹사이트에 **악성 스크립트를 삽입**하여 사용자 브라우저에 실행 | `<script>` 공격, 게시판, 쿠키 탈취 |
| **SQL 인젝션 (SQL Injection)**     | 입력값에 SQL문 삽입 → **DB 명령어 실행**         | `' OR '1'='1`, 비밀번호 우회    |
| **버퍼 오버플로우 (Buffer Overflow)**  | **메모리 범위 초과 입력으로 코드 실행**             | 스택 덮기, shell 실행           |
| **랜섬웨어 (Ransomware)**           | 파일을 **암호화하고 금전 요구**                  | 파일 잠금, 복호화 키 요구           |
| **제로 데이 공격 (Zero-day Attack)**  | **보안 패치가 안 된 취약점**을 악용               | 개발사도 모르는 미공개 취약점          |
| **워터링 홀 공격**                    | 타깃 사용자가 자주 가는 사이트를 감염시킴              | 방문만 해도 감염                 |
| **백도어 (Backdoor)**              | **숨겨진 비인가 접근 경로**                    | 우회, 몰래 접속                 |
| **디도스 (DDoS)**                  | 여러 대의 PC로 **서비스 마비**시키는 공격           | 분산 공격, 트래픽 폭탄             |
